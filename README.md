# Benchmarking and Performance

This repository is a working space for building a literature survey and project direction around benchmarking modern computing systems. The focus is on how performance is measured, represented, compared, and made reproducible in an academic context.

## Early Questions

- What makes a benchmark scientifically useful rather than only a ranking?
- How do researchers choose workloads that represent real systems and users?
- Which metrics matter most: throughput, latency, energy, cost, accuracy, fairness, or stability?
- How should benchmark results report uncertainty, variance, and experimental conditions?
- What standards do established benchmark suites use for reproducibility and fairness?

## Survey Scope

The survey will start broad and then narrow as the project becomes clearer. Initial areas of interest include:

- Benchmarking methodology and experimental design
- Systems performance measurement
- Reproducibility and reliability of benchmark results
- Benchmark suites such as SPEC, TPC, and MLPerf
- Domain-specific benchmarking in HPC, databases, cloud systems, and machine learning systems
- Data representation and reporting practices for performance results

## Project Direction

The goal for now is not to define a final benchmark, but to understand how strong benchmarking work is framed. This README will evolve as the literature survey grows and the research questions become sharper.
